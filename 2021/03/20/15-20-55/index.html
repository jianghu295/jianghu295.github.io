类集框架

集合框架

  • Java的集合框架使你的程序处理对象组的方法标准化
  • 集合框架被设计的目的：
    • 这种框架是高性能的。对基本集合类（动态数组，链接表，树和散列表）的实现是高效率的
    • 框架必须允许不同类型的集合以相同的方式和高度互操作方式操作
    • 集合是容易扩展和或修改的
  • Java平台提供了一个全新的集合框架。“集合框架”主要由一组用来操作对象的接口组成,位于java.util包下。不同接口描述一组不同数据类型
  • Collection接口——定义了存取一组对象的方法，将对象以一个接一个的方式存储，其子接口Set和List分别定义了具体的存储方式
    • Set中的数据对象没有顺序且不可以重复
    • List中的数据对象有顺序且可以重复

1.1 List接口

  • List接口是Collection的子接口，实现List接口的容器类中是有顺序的，且可以重复的
  • 类似数组，但可以动态增长，查找元素效率高，插入删除元素效率低，因为会引起其他元素位置的改变。
  • List接口的容器类有ArrayList和LinkedList

1.2.1 HashSet

无实现了Set接口，创建的集合使用散列表进行存储。

HashSet没有定义任何新的方法。

无序的，若需要排序存储，则采用另一种集合类TreeSet

1.2.2 TreeSet

此类实现 Set 接口，该接口由 TreeMap 实例支持。此类保证排序后的 set 按照升序排列元素，根据使用的构造方法不同，可能会按照元素的自然顺序 进行排序（参见 Comparable），或按照在创建 set 时所提供的比较器进行排序。

如果自定义对象，实现Comparable接口，重写compareTo方法

2 Map(映射)

  • 映射（Map）是一个存储关键字和值的关联或者说是关键字/值对的对象。给定一个关键字，可以得到它的值。关键字和值都是对象。关键字必须是唯一的。但值可以重复的。
  • Map接口映射唯一关键字到值。它使按照关键字(key)进行存储和检索的。存储时采用什么关键字将来检索时也用什么关键字
  • Map接口的实现类
    • HashMap
    • TreeMap
方法描述
void clear( )从调用映射中删除所有的关键字/值对
boolean containsKey(Object k)如果调用映射中包含了作为关键字的k，则返回true；否则返回false
boolean containsValue(Object v)如果映射中包含了作为值的v，则返回true；否则返回false
Object get(Object k)返回与关键字k相关联的值
boolean isEmpty( )如果调用映射是空的，则返回true；否则返回false
Object put(Object k, Object v)将一个输入加入调用映射，覆盖原先与该关键字相关联的值。
关键字和值分别为k和v。如果关键字已经不存在了，则返回null；
否则，返回原先与关键字相关联的值
第二次赋值就是修改，得到的是上一次值
Object remove(Object k)删除关键字等于k的输入
int size( )返回映射中关键字/值对的个数
keySet()返回此映射中所包含的键的 set 视图。

2.3 HashTable与HashMap

历史原因:Hashtable是基于陈旧的Dictionary类的，HashMap是Java 1.2引进的Map接口的一个实现

同步性:Hashtable是线程安全的，也就是说是同步的，而HashMap是线程序不安全的，不是同步的

值：只有HashMap可以让你将空值作为一个表的条目的key或value ，map键值都可以为空，Hashtable键值都不可以为空
