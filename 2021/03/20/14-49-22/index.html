第八章 继承和多态

8.1 父类和子类

继承使得你可以定义一个通用的类（即父类）, 之后扩充该类为一个更加特定的类（子类）

注意

  • 子类并不是父类的一个子集。实际上，一个子类通常比它的父类包含更多的信息和方法
  • 父类中的私有数据域在该类之外是不可访问的。除非父类定义了公共的访问器/修改器
  • 某些程序设计语言允许从几个类派生出一个子类。这种能力成为多重继承。但是在** java 中不允许多重继承的。这种限制成为单一继承**。

8.2 super

关键字* super *指代父类，可以用于调用父类中的普通方法和构造方法。

最好能为每个类提供一个无参构造方法，以便对该类进行扩展，同时避免错误

8.3 方法重写

要重写一个方法，需要在子类中使用和父类一样的签名以及一样的返回值类型来对该方法进行定义

  • 仅当实例方法是可访问时，它才能被覆盖。
  • 与实例方法一样，静态方法也能被继承。但是，静态方法不能被覆盖。如果父类中定义的静态方法在子类中被重新定义，那么在父类中定义的静态方法将被隐藏。可以使用语法父类名。静态方法名调用隐藏的静态方法。

8.4 方法重写和重载

重载意味着使用同样的名字但是不同的签名来定义多个方法

重写意味着在子类中提供一个对方法的重新实现

  • 重写发生在通过继承而相关的不同类中；重载可以发生在同一个类中，也可以发生在由于继承而相关的不同类中
  • 重写具有同样的签名和返回值类型；重载具有同样的名字，但是不同参数列表

8.5 多态（359）

多态意味着父类的变量可以指向子类对象

使用父类对象的地方都可以使用子类的对象。这就是通常所说的多态. 简单来说，多态意味着父类型的变量可以引用子类型的变量。

8.6 动态绑定

方法可以沿着继承链的多个类中实现。JVM 决定运行时调用哪个方法。

声明类型和实际类型。在子类重写父类中的方法，调用哪个方法由实际类型决定。

上转型对象访问的是父类的属性及子类的重写方法

匹配方法的签名和绑定方法的实现是两个不同的问题。引用变量的声明类型决定了编译时匹配哪个方法。在编译时，编译器会根据参数类型、参数个数和参数顺序找到匹配的方法。一个方法可能在沿着继承链的多个类中实现。Java 虚拟机在运行时动态绑定方法的实现，这是由变量的实际类型决定的。

8.7 对象转换和* instanceof *运算符

对象的引用可以类型转换为对另外一种对象的引用，这称为对象转换

将一个子类的实例转换为一个父类的变量称为向上转换，以为子类的实例永远是它的父类的实例。

把一个父类的实例转换为它的子类变量称为向下转换，必须使用转换记号“（子类名）”进行显示转换

如果父类对象不是 子类的一个实例，就会出现一个运行异常* ClassCastException*

为了能够进行通用程序设计，一个好的经验是把变量定义为父类型，这样 ，它就可以接收任何子类型的值

*instanceof *运算符：确保源对象是否是目标类的实例

注意

对象成员访问运算符 (.) 优先于类型转换运算符

第九章 抽象类和接口

父类中定义了相关子类中的共同行为。

接口可以用于定义类的共同行为（包括非相关的类）

9.1 抽象类

抽象类不可以用于创建对象。

抽象类可以包含抽象方法，这些方法将在具体的子类中实现。

说明

  • 一个包含抽象方法的类必须声明为抽象类，抽象类不一定包含抽象方法，抽象类的实现由子类实现。抽象方法是非静态的。
  • 抽象类不能使用* new *操作符来初始化的。但是，可以定义它的构造方法。
  • 即使子类的父类是具体的，这个子类也可以是抽象的。例如，*object *类

9.2 接口

接口是一种与类相似的结构，只包含常量和抽象方法。

接口在许多方面都与抽象类很相似，但是它的目的是指明相关或者不相关类的多个对象的共同行为。例如，使用正确的接口，可以指明这些对象是可比较的、可食用的，以及可克隆的。

在 Java 中，接口被看做是一种特殊的类。

接口中所有的数据域都是* public static final，而且所有的方法都是 public abstract*，所以 Java 允许忽略这些修饰符。

9.3 Comparable 和 Comparator

9.3.1 Comparable（437）

*Comparable 接口定义了 comparaTo *方法，用于比较对象。

*comparaTo 方法判断这个对象相对于给定对象 o 的顺序，并且当这个对象小于、等于或大于给定兑现 o *时，分别返回负整数、0 或正整数。

所有* Comparable 对象都有 comparaTo 方法，如果对象是 Comparable 接口类型的实例的话，Java API 中的 java.util.Arrays.sort(Object[]) 方法就可以使用 comparaTo *方法来对数组中的对象进行比较和排序。

sort 方法只能对实现接口 Comparable 的类进行排序，可以定义一个新的类实现 Comparable，且这个新类的实例是可比较的。

第十章 异常处理

异常处理使得程序可以处理非预期的场景。

10.1 异常处理概述

异常是从方法抛出的。方法的调用者可以捕获以及处理该异常。

throw 语句类似于方法的调用，但不用于方法的是，他调用的是* catch 块。从某种意义上讲，catch 块就像带参数方法的方法定义这些参数匹配抛出的值得类型。但是，它不像方法，在执行完 catch 块之后，程序控制不返回到 throw 语句；而是执行 catch *块后的下一条语句。

一个异常可能是通过* try 块中的 throw *语句直接抛出，或者调用一个可能会抛出异常的方法而抛出。

10.2 异常类型

异常是对象，而对象都采用类来定义。异常的根类是* java.lang.Throwalbe*。

Throwable 是所有异常类的根。所有的异常类都直接或者间接地继承自 Throwable。可以通过继承* Exception 或者 Exception *的子类来创建自己的异常类。

异常类可分为三种主要类型

  • 系统错误 (error)
    • *java 虚拟机抛出，error *类表示。
    • 此类错误通常不会设计程序去捕获处理。
  • 异常（Exception）：一般性的错误，通常 a) 捕获后可做处理，以确保程序继续运行
    • 其他异常类：必须处理，否则无法运行
    • RuntimeException（发生频率比较高的异常，可以处理也可以不处理）
      • ArithmeticException：数学类异常
      • ArrayIndexOutOfBoundsException：数组下标越界异常
      • NullPointerException：空指针异常
      • ClassNotFoundException：找不到类异常
      • NumberFormatException：数字格式异常

10.3 关于异常处理的更多知识

异常的处理器是通过从当前的方法开始，沿着方法调用链，按照异常的反向传播方向找到的。

  • 声明一个异常
  • 抛出一个异常
  • 捕获一个异常

如果抛出多个异常

如果方法没有在父类中声明异常，那么就不能在子类中对其进行继承来声明异常。

Java 强迫程序员处理必检异常（*Error 或 Runtime Exception *之外**的异常）。如果方法声明了一个必检异常，就必须在* try-catch 块中调用它，或者在调用方法中声明要抛出异常。*

例如，p1 调用方法 p2，而 p2 可能会抛出一个必检异常。

使用同样的代码处理多个异常（jdk7 的多捕获特征）
