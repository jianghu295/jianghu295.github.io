设计模式

Author: JiangHural Date: 三月 26, 2021  8:46:31 Category: Java

设计模式的六大原则

对修改关闭，对扩展开放

创建者模式

单例模式 (Singleton)

java 单例模式：饿汉式与懒汉式

单例模式，任何模式得到的对象都是同一个对象

  • 构造方法私有化
  • 提供全局的访问点，获得对象
  • 有一个本类的属性

类里面有代表自己特征的属性不能用单例模式。如：身高，体重。..
数据库连接—>单例模式

行为模式

命令模式

通过实际的需求调用执行的对象，至于执行的细节，全部封装在定义好的方法中。

要实现命令模式，首先需要一个所有命令通用的接口，其中定义了所有命令所必须实现的方法。

  • 创建一个通用接口，定义一个通用的方法
  • 创建一个调用类，用于添加命令和执行通用方法
  • 创建多个实现了通用接口的类，使通用方法完成各自所需的功能。
  • 使用调用类。

策略模式

Java设计模式之策略模式

现实中我们可以对同一事件采取不同的策略，通过不同的策略可以得到不同的结果。我们可以将策略加以封装为一个对象，而不是将策略写死在某个类中，是的策略可以独立于客户端，随时改变策略的内容，都不会对客户端程序造成影响。

在软件构建过程中，某些对象使用的算法可以多种多样经常变化，如果将这些算法都编码到对象中，将会使对象变得异常复杂。而且有时候支持不使用的算法也是一个性能负担。

如果想要在运行时根据需要透明地更改对象的算法，可以将对象与算法本身解耦，从而实现目的。

  • 创建一个策略抽象类
  • 创建一个调用类
  • 创建不同的策略继承抽象类
  • 测试程序
